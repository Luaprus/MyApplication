# HarmonyOS 健康记录 App 项目说明文档

## 项目一句话简介

这是一个基于 HarmonyOS ArkTS 的健康记录 App，围绕饮食、饮水、体重、运动、AI 健康助手和趋势周报，帮助用户完成日常健康数据记录与智能反馈。

## 项目背景和用途

日常减脂、控糖或保持运动习惯时，用户经常遇到三个问题：记录分散、反馈滞后、坚持成本高。本项目把餐食、饮水、体重、运动和打卡集中到同一个移动端入口，并通过趋势日历、周报分析和 AI 问答把“记录数据”转化成“可行动建议”。

项目适合用于课堂展示一个完整的 HarmonyOS 应用实践：既有 ArkTS 客户端，也有 Node.js 后端；既有页面交互，也有状态持久化、用户认证、远程同步和第三方 AI/食物库代理。

## 核心功能列表

- 账号登录与注册：支持本地账号、演示账号、首次资料初始化。
- 首页健康记录：展示打卡、体重管理方案、饮食热量、今日完成度、智能提醒和趋势日历。
- 餐食记录：支持早餐、午餐、晚餐、加餐分类，提供食物库搜索、手动添加、常用食物和拍照识别入口。
- 饮水记录：根据体重计算饮水目标，展示今日饮水进度和智能提醒。
- 运动记录：按热门、步行、跑步、骑行、球类、健身、拉伸分类，记录运动消耗。
- 体重管理：展示初始体重、当前体重、目标体重、本周减重和趋势曲线。
- AI 健康助手：提供快捷提问、聊天历史、周报解读入口，结合饮食/体重/运动上下文生成建议。
- 趋势日历和周报：按日期回看摄入、饮水、运动和体重；在个人中心生成本周概览。
- 个人中心：展示会员等级、积分、健康分、目标、数据报告、周报、账号安全和反馈入口。

## 页面/模块结构

项目主体位于 `entry/src/main/ets`：

- `pages/Index.ets`：主页面入口，负责全局状态、Tabs、页面编排和主要事件分发。
- `pages/modules/AuthModule.ets`：登录、注册、演示账号入口。
- `pages/modules/RecordModule.ets`：健康记录首页容器，承载打卡、体重、饮食、完成度和趋势等区块。
- `pages/modules/AiModule.ets`：AI 健康助手聊天页、快捷提问和历史清空。
- `pages/modules/ProfileModule.ets`：个人中心页面容器。
- `models/`：业务计算和展示模型，包括饮水目标、体重计划、餐食热量、运动统计、趋势日历、周报、AI prompt 构建等。
- `services/`：后端 API、认证、远程状态同步、DeepSeek、豆包视觉识别、薄荷食物库代理。
- `resources/base/media/`：底部导航、功能图标、饮品图标、餐食/运动图标和启动图标。

配套后端位于 `backend/`：

- `backend/src/index.js`：Express API 路由入口。
- `backend/src/lib/users.js`：本地用户、演示账号、token 签发与校验。
- `backend/src/lib/user-state.js`：按用户隔离保存远程状态。
- `backend/src/services/deepseek.js`：AI 对话代理。
- `backend/src/services/doubao.js`：图片识别食物热量代理。
- `backend/src/services/boohee.js`：薄荷食物库搜索代理。

## 用户使用流程

1. 用户进入登录页，可以注册账号，也可以点击“演示账号”快速体验。
2. 登录后进入记录首页，先看到打卡、体重管理方案和饮食热量概览。
3. 用户可以记录餐食、饮水、运动和体重，也可以查看今日完成度。
4. 首页下滑后可以看到智能提醒、趋势日历、日期详情和目标动态调整。
5. 用户切换到 AI 页，可以用快捷问题或自由输入咨询健康建议。
6. 用户切换到“我的”，查看个人资料、健康分、积分、目标、周报分析和账号功能。
7. 在周报分析中，用户可以查看本周总览、体重趋势，并让 AI 解读周报。

## 技术架构说明

客户端采用 HarmonyOS Stage 模型，`module.json5` 声明了 `EntryAbility`，入口页面来自 `resources/base/profile/main_pages.json` 中的 `pages/Index`。应用目标 SDK 为 HarmonyOS `6.0.1(21)`，设备类型为 `phone`。

客户端通过 `BackendApiService.ets` 访问后端，模拟器默认地址是 `http://10.0.2.2:5200/api`，并在该地址不可用时尝试回退到 `127.0.0.1` 和 `localhost`。网络请求配置了不同超时策略，AI 识图和 AI 聊天比普通接口有更长读取超时。

后端是 Express 服务，默认监听 `5200` 端口。认证接口负责注册、登录、资料更新和密码修改；业务接口包括 AI 聊天、图片识别食物热量、食物搜索和用户状态同步。第三方 API Key 统一放在后端 `.env` 中，避免暴露在 HAP 安装包里。

状态层分为本地持久化和远程同步。本地使用 HarmonyOS preferences 保存基础状态；登录用户可通过 `/api/user/state` 将健康记录同步到后端。离线或后端不可用时，客户端内置了离线演示模式，确保核心页面仍可展示。

## 关键源码文件说明

- `entry/src/main/ets/pages/Index.ets`：项目最大、最核心的页面文件，包含认证状态、健康数据状态、弹层开关、Tabs、打卡、餐食、运动、体重、AI、个人中心等主要交互。
- `entry/src/main/ets/models/IndexSeedData.ets`：定义默认 AI 消息、AI 快捷建议、个人中心菜单、餐食预设、运动预设、饮品预设。
- `entry/src/main/ets/models/FeatureInsights.ets`：生成趋势日历、智能提醒、目标调整、成就汇总、CSV 导出等高价值分析结果。
- `entry/src/main/ets/models/AiPromptBuilders.ets`：把当前健康上下文组织成 AI 系统提示词和周报提示词。
- `entry/src/main/ets/services/BackendApiService.ets`：封装 HTTP 请求、token header、超时配置、模拟器地址回退和错误文案。
- `entry/src/main/ets/services/AuthService.ets`：封装登录、注册、资料更新、首次资料初始化和改密码。
- `entry/src/main/ets/services/AppStateStore.ets`：本地 preferences 持久化。
- `backend/src/index.js`：后端 API 路由集中定义，包含健康检查、认证、状态同步、AI 和食物库接口。
- `backend/src/lib/users.js`：演示账号、用户注册登录、token 签发校验和用户资料更新。

## 项目亮点

- 功能链路完整：从登录、记录、趋势、个人中心到 AI 助手，具备一个健康管理 App 的完整体验。
- 展示效果成熟：页面视觉统一，移动端信息层级清楚，适合课堂展示。
- 数据驱动反馈：不是只做表单记录，而是用完成度、智能提醒、趋势日历和周报把记录转成反馈。
- AI 能力接入方式合理：客户端不直接保存 DeepSeek、豆包、薄荷食物库密钥，而是通过后端代理。
- 可离线演示：即使后端或第三方 API 不完整，也有演示模式和预设数据支撑主页面展示。
- 工程拆分有意识：虽然 `Index.ets` 仍然很大，但已拆出多个 models 和 modules，便于说明重构思路。
